Vorinthex AI Local Design Engine — Full Build Prompt
Goal
Build a complete local Bun-based design engine for Vorinthex AI that uses the OpenAI Responses API and image generation/editing models to create, iterate, review, lock, version, export and manage brand assets locally.
This must be a full working tool, not an MVP.
The tool must let me define a global design system once, keep it persistent, lock assets permanently, generate both solid-background and transparent PNG outputs, continue from previous generations, review outputs, compare versions, and maintain a registry of all design assets.
The first target use case is creating Vorinthex AI logos and icon assets at exactly 1024px × 1024px.

Lock Registry
Create `registry/locks.json`.
Locks are critical.
When an asset is locked:
Its silhouette must not be redesigned
Its core geometry must not change
Future improvements must preserve it
The lock must be included in future prompts
The CLI must warn before modifying it
The CLI must require explicit unlock before major redesign
Example:
```json
{
  "vorinthex-ai": {
    "assetId": "master-brand-vorinthex-ai",
    "lockedVersionId": "v7",
    "lockedAt": "2026-07-06T12:00:00.000Z",
    "lockLevel": "strict",
    "rules": [
      "Keep circular chrome ring",
      "Keep central downward triangular nexus symbol",
      "Keep minimal black inner field",
      "No extra stars or glitter",
      "No text inside icon",
      "Preserve silhouette exactly"
    ]
  }
}
```
Lock levels:
```txt
soft    = allow changes, preserve concept
medium  = preserve silhouette and major geometry
strict  = preserve silhouette, layout, proportions and identity
frozen  = no image regeneration except background/export processing
```
