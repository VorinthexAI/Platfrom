# Phase 8 — Content Pipeline: ROLE_STORY & Output Ingestion

## Goal

Build the content-generation pipeline as Agent/Manager/Role/Task data, using existing Actions.

## Shape

- `AGENT_CMO` owns `MANAGER_CMO_CONTENT`.
- `MANAGER_CMO_CONTENT` owns `ROLE_STORY` and `ROLE_INGEST_OUTPUTS`.
- `ROLE_STORY` runs the content-generation task chain.
- `ROLE_INGEST_OUTPUTS` ingests produced artifacts when the Manager decides ingestion should run.

## Tasks

- `TASK_PREPARE_MISSION`: `ACTION_FETCH_S3_CONTEXT` → `ACTION_AI_GENERATE` → `ACTION_ENRICH`
- `TASK_DISCOVER_SUBREDDITS`: `ACTION_RESEARCH` → `ACTION_COUNCIL`
- `TASK_SCAN_SUBREDDIT_TRENDS`: `ACTION_RESEARCH` → `ACTION_COUNCIL`
- `TASK_FETCH_BLOG_CONTEXT`: `ACTION_RESEARCH` → `ACTION_COUNCIL`
- `TASK_DISCOVER_TRENDS`: `ACTION_RESEARCH` → `ACTION_COUNCIL`
- `TASK_GENERATE_CATEGORIES`: `ACTION_AI_GENERATE`
- `TASK_MERGE_CATEGORIES`: `ACTION_AI_GENERATE`
- `TASK_GENERATE_TOPICS`: `ACTION_AI_GENERATE`
- `TASK_MERGE_TOPICS`: `ACTION_AI_GENERATE`
- `TASK_GENERATE_HOOKS`: `ACTION_AI_GENERATE`
- `TASK_GENERATE_ASSET_CONCEPTS`: `ACTION_AI_GENERATE`
- `TASK_GENERATE_HASHTAGS`: `ACTION_AI_GENERATE`
- `TASK_WRITE_POST`: `ACTION_FETCH_S3_CONTEXT` → `ACTION_ENRICH` → `ACTION_GET_POST_SCHEMA` → `ACTION_AI_GENERATE`
- `TASK_INGEST_OUTPUT_ARTIFACT`: `ACTION_SAVE_OUTPUT` → `ACTION_EMBED`

## Wiring

Use `ACTION_CREATE_AGENT`, `ACTION_CREATE_MANAGER`, `ACTION_CREATE_ROLE`, `ACTION_CREATE_TASK`, `ACTION_APPEND_MANAGER_TO_AGENT`, `ACTION_APPEND_ROLE_TO_MANAGER`, and `ACTION_APPEND_TASK_TO_ROLE`.

## Success criteria

- `ROLE_STORY` runs its tasks in order.
- `MANAGER_CMO_CONTENT` receives completion and decides whether `ROLE_INGEST_OUTPUTS` runs.
- `ROLE_INGEST_OUTPUTS` serializes ingestion and writes output relations after the full batch is visible.
