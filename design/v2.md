## 13. Live, Teaser, Locked, and Hidden States

Every object must have `isLive` and `visibility`.

These fields determine what users can see and interact with.

### `isLive: true`

The product/object is launched.

Render full content.

Allow full navigation.

Allow full CTA.

Allow SEO indexing if `seo.indexable: true`.

### `isLive: false` with `visibility: 'teaser'`

The product/object is not launched, but can be shown publicly.

Render:

- name
- logo
- short teaser
- launch date or coming soon
- waitlist CTA
- dark/grey/locked visual treatment

Hide:

- deep product details
- unreleased features
- pricing if not ready
- internal strategy

### `isLive: false` with `visibility: 'locked'`

The product/object exists in the galaxy but is visually locked.

Render as a dark object.

Allow hover/click to reveal minimal teaser.

Do not show full product content.

### `visibility: 'hidden'`

Do not render publicly.

Do not include in sitemap.

Do not include in llms.txt.

Do not include in public route generation.

## 17. Content Must Come From Registry

All copy must come from registry.

This includes:

- product name
- product tagline
- product short description
- product long description
- CTA labels
- launch date
- status labels
- page headlines
- product cards
- modal text
- hover labels
- SEO title
- SEO description
- AEO answer snippets
- llms.txt summaries
- OpenGraph data
- schema data

Components should be content-agnostic.

For example, this is wrong:

```tsx
<h2>Archive</h2>
<p>Your second brain.</p>
```

This is correct:

```tsx
<h2>{entity.name}</h2>
<p>{entity.shortDescription}</p>
```

This is wrong:

```tsx
if (slug === 'archive') return <ArchivePanel />
```

This is correct:

```tsx
return <EntityPanel entity={entity} />
```

Special visual components are allowed, but they must still receive content from the registry.

# V4 ADDENDUM — Dormant States, Visibility, Cursor Interaction Rules

This addendum extends the existing Vorinthex AI galaxy landing page prompt.
It must be treated as a non-negotiable implementation requirement.

The goal is to make the galaxy feel alive even when products, capabilities, orchestrators, moons, asteroids, and collectibles have not launched yet.

Nothing should feel broken.
Nothing should feel missing.
Nothing should feel accidentally disabled.

Unreleased objects should feel dormant, mysterious, premium, and discoverable.

---

## 47. Discovery & Unlock States

The galaxy should never feel empty.

Products, orchestrators, capabilities, moons, asteroids, and future resources that have not yet launched must not disappear completely unless their registry visibility explicitly says they are hidden.

The registry should support multiple states:

```ts
export type EntityVisibility =
  | 'live'
  | 'dormant'
  | 'teaser'
  | 'locked'
  | 'hidden';
```

Recommended meaning:

- `live`: fully released, fully interactive, full content visible.
- `dormant`: visible but not launched yet; can be explored but not purchased/activated.
- `teaser`: visible with limited teaser content.
- `locked`: visible, but gated behind invite, waitlist, account, or launch milestone.
- `hidden`: not rendered publicly and not included in public navigation or public SEO.

The visual system must distinguish these states clearly while preserving the premium aesthetic.

---

## 48. Dormant State Visual Rules

A dormant object should remain visible.

Never render unreleased resources completely black.
Never make them so dark that users cannot see what they are.
Never hide them unless `visibility === 'hidden'`.

Dormant objects should look like sleeping celestial bodies waiting to awaken.

Use:

- desaturated chrome
- dark silver
- graphite metal
- subtle glass reflections
- low-intensity emissive glow
- soft rim light
- reduced bloom
- reduced animation speed
- slightly transparent atmosphere layer
- subdued logo visibility
- muted orbit ring
- lower contrast than live objects

Avoid:

- pure black materials
- invisible labels
- disabled-looking UI
- harsh grayscale blocks
- ugly opacity fade-outs
- red warning states
- broken placeholder icons
- generic lock icons everywhere

The dormant state should communicate:

> This exists. It is coming. You can explore it. It has not awakened yet.

---

## 49. Live State Visual Rules

When an entity has:

```ts
isLive: true
visibility: 'live'
```

it should render as active.

Enable:

- full chrome material
- stronger reflections
- stronger but tasteful bloom
- active orbit animation
- full label opacity
- active hover states
- active CTA buttons
- videos
- screenshots
- full description content
- indexable SEO pages if configured

The live state should not require rewriting the component.

The only source of truth must be the registry.

Changing `isLive` from `false` to `true` should be enough to transform a dormant object into an active product, capability, or orchestrator.

## 52. Hover Philosophy

Hovering an object should feel physical.

Every interaction should feel deliberate, quiet, expensive, and smooth.

Avoid playful arcade-style animation.
Avoid cheap glow spam.
Avoid colorful hover effects.

The user should feel like they are interacting with a luxury instrument.

### Planet hover

- slightly scale up
- orbit ring brightens
- subtle magnetic pull toward cursor
- label sharpens
- rim light intensifies
- nearby dust particles react subtly

### Capability asteroid hover

- asteroid rotation slows
- logo turns toward camera
- title fades in
- small chrome label appears
- orbit path becomes visible

### Orchestrator moon hover

- moon rotates toward camera
- orbit line brightens
- role and scope appear
- Command planet subtly glows in response

### Button hover

- subtle scale
- chrome reflection sweep
- shadow deepens
- border becomes slightly brighter
- no neon color accents

### Card hover

- shallow lift
- slight perspective shift
- soft metallic edge highlight
- no aggressive color changes

## 55. Registry-Driven Rendering Logic

Components must read state from the singleton registry.

Example:

```ts
function getEntityRenderState(entity: RegistryEntity) {
  if (entity.visibility === 'hidden') return 'hidden';
  if (entity.isLive && entity.visibility === 'live') return 'active';
  if (entity.visibility === 'locked') return 'locked';
  if (entity.visibility === 'teaser') return 'teaser';
  return 'dormant';
}
```

The renderer should use this derived state to decide:

- material intensity
- label opacity
- whether CTAs are active
- whether full content is shown
- whether SEO page is indexable
- whether product can be deep-linked
- whether teaser modal appears
- whether claim/reward interactions are enabled
